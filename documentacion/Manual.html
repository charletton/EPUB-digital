1. Functions

  1. Funcion para anadir
  2. Funciona para anadir libros

1.1. Ejemplo de uso functions ’add author’

Como primer argumento de la funcion se agrega el nombre del autor. El segundo argumento tiene que ser una descripcion breve para tener una vision general del mismo 

-- Buscando por categoria y agregando
CALL fn_add_autor('Miguel de cervantes', 'Descripcion');

1.2. Ejemplo de uso funcion ’add book’

Para anadir un libro, necesitamos una serie de argumentos

  • Nombre de del libro
  • ID del autor
  • ID del libro
  • Tomando como ejemplo el autor anadido de la funcion fnaddauthor, consultaremos los autores para obtener su respectiva id
SELECT * FROM AUTOR;
  1. Luego consultaremos la id de la categoria que pertenece (por ejemplo ficcion)
SELECT * FROM CATEGORIA;
  1. Luego de obtener todos los parametros suficientes para anadir al autor, podemos finalmente llamar a la funcion y pasar los respectivos parametros
CALL fn_add_book('Don quijote', reemmplazar_por_id_autor, reemplazar_id_categoria);

2. Stored procedures

  • Procedure para calcular ventas por mes
  • Procedures para buscar libros filtrando por autor y/o categoria

2.1. Ejemplo de uso ’ventas por mes’

Ventas por mes es un procedure sencillo que simplemente calcula las ventas por mes 

CALL Ventas_Por_Mes();

2.2. Ejempl de uso ’buscar por autor o categoria’

-- Buscar libros por autor
CALL Buscar_Libros_Por_Autor_O_Categoria('Marco Aurelio', '');
-- Buscar libros por categoria
CALL Buscar_Libros_Por_Autor_O_Categoria('', 'Ficción');
-- Ver ventas por mes

3. Views

3.1. View para el total gastado del usuario

Esta view para observar a los clientes mas frecuentes 

SELECT * FROM vw_Total_gastado_usuario;

Se puede usar de forma practica para ver de forma descendente el total gastado por los usuarios 

SELECT * FROM vw_Total_gastado_usuario ORDER BY total_gastado DESC;

3.2. View para ver autores por categoria

View que a diferencia del filtro de libros por autor y categoria, te permite visualizar a que categoria se dedica cada autor que se encuentra cargado en la base de datos 

SELECT * FROM vw_autores_por_categoria;

3.3. View para ver libros comprados por usuario

View simple, muestra que libros compro cada usuario 

select *  from vw_Libros_Comprados_Por_Usuario;

3.4. View para ver comentarios por usuario

View que muestra comentarios echos por usuarios 

SELECT *  FROM vw_Libros_Comprados_Por_Usuario;

4. Triggers

4.1. Trigger ’calcular pedidos por usuario’

Este trigger simplemente por cada pedido que haga un usuario, va a aumentar en 1 la columna totalpedidos del correspondiente usuario

4.2. Trigger ’calcular libros por autor’

Cada autor tiene su cantidad de libros que se encuentran en la base de datos de la empresa, por cada libro que se inserte en la base, su contador aumentara en 1
